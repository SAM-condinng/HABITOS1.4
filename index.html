📱 Install as App (PWA)

To make HabitOS installable on any device, you need a manifest.json file placed in the same folder as your index.html.

Setup steps:
1. Download both files above
2. Place manifest.json and sw.js next to index.html
3. Serve from a web server (not file://)
4. Chrome/Edge: install icon in address bar
5. iOS Safari: Share → "Add to Home Screen"
⊕ Connect your project

Paste your Supabase Project URL and anon key from Settings → API.

◈ Database schema (updated with time field)
-- habits CREATE TABLE habits ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, name TEXT NOT NULL, emoji TEXT DEFAULT '🎯', created_at TIMESTAMPTZ DEFAULT now() ); -- habit_logs CREATE TABLE habit_logs ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, habit_id UUID REFERENCES habits(id) ON DELETE CASCADE, completed_date DATE DEFAULT current_date, UNIQUE(habit_id, completed_date) ); -- todos (with time field) CREATE TABLE todos ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, text TEXT NOT NULL, priority TEXT DEFAULT 'med', done BOOLEAN DEFAULT false, date DATE DEFAULT current_date, task_time TIME, -- NEW: assigned time e.g. '09:30' created_at TIMESTAMPTZ DEFAULT now() ); -- habit_notes (journal entries) CREATE TABLE habit_notes ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, habit_id UUID REFERENCES habits(id) ON DELETE CASCADE, note_date DATE DEFAULT current_date, text TEXT NOT NULL, ts BIGINT, user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE, created_at TIMESTAMPTZ DEFAULT now(), CONSTRAINT habit_notes_unique UNIQUE(habit_id, note_date, user_id) );
▣ Add user_id columns
ALTER TABLE habits ADD COLUMN user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE; ALTER TABLE habit_logs ADD COLUMN user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE; ALTER TABLE todos ADD COLUMN user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE; -- habit_notes already includes user_id in CREATE TABLE above
▣ RLS policies (auth-based)
ALTER TABLE habits ENABLE ROW LEVEL SECURITY; ALTER TABLE habit_logs ENABLE ROW LEVEL SECURITY; ALTER TABLE todos ENABLE ROW LEVEL SECURITY; ALTER TABLE habit_notes ENABLE ROW LEVEL SECURITY; -- Drop old open policies if they exist DROP POLICY IF EXISTS "open" ON habits; DROP POLICY IF EXISTS "open" ON habit_logs; DROP POLICY IF EXISTS "open" ON todos; DROP POLICY IF EXISTS "open" ON habit_notes; -- Each user sees only their own rows CREATE POLICY "own" ON habits FOR ALL USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id); CREATE POLICY "own" ON habit_logs FOR ALL USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id); CREATE POLICY "own" ON todos FOR ALL USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id); CREATE POLICY "own" ON habit_notes FOR ALL USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);
◈ Enable Email Auth

In your Supabase project go to Authentication → Providers → Email and make sure it is enabled. That is all — no extra tables needed. Supabase manages the auth.users table automatically.
